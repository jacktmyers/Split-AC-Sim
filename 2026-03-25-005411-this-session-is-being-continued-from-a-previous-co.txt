❯ How could I get around using the gui to reference m_selectedParticles?        

  Searched for 1 pattern (ctrl+o to expand)

● m_selectedParticles lives entirely inside Simulator_GUI_imgui, and since
  you're running without a GUI there's no meaningful notion of "selected
  particles". The cleanest options are:

  1. Skip selected particle rendering entirely — which is effectively what the
  early return does now. If there's no GUI, there's no one to select particles,
  so nothing to render.
  2. Own the selection state yourself — declare std::vector<std::vector<unsigned
   int>> g_selectedParticles; as a global in Custom.cpp and pass it into
  CustomRender instead of gui. You'd populate it yourself (e.g.,
  programmatically select particles for debugging).
  3. Pass gui as an optional — only enter the selected-particles block when gui 
  != nullptr, skipping it otherwise (similar to option 1 but without the blanket
   early return).

  Since you have no GUI and presumably no input for selecting particles, option
  1 (the current early return approach) is the right call — the block is dead
  code without a GUI.
